# 🏷️ Tag Manager

Веб-приложение на **Streamlit** для генерации детальных капшенов/тегов к изображениям
с помощью локального мультимодального LLM через **OpenAI-совместимый API**
(эндпоинт `/v1/chat/completions` с поддержкой `image_url`).

Работает с любым локальным сервером, отдающим OpenAI-совместимый API:
**llama.cpp (`llama-server`)**, **oobabooga / text-generation-webui**, LM Studio и т.п.

Заточено под подготовку датасетов для обучения **LoRA**: строго последовательная
обработка папки с изображениями, для каждой картинки пишется одноимённый `.txt`.

## Возможности

- Выбор папки с изображениями (текстовый путь **или** кнопка «Обзор» через системный диалог), рекурсивно или только текущий уровень.
- Поддержка `.jpg`, `.jpeg`, `.png`, `.webp`; для каждой картинки пишется одноимённый `.txt`.
- Режимы обработки: докачка (только не сделанные этим приложением), все файлы с перезаписью, только без капшенов, пропуск по дате `.txt`.
- Строго **последовательная** обработка (одно изображение за раз).
- Отправка изображения в base64 (multimodal `image_url`).
- Проверка качества капшена и авто-retry (до 3 раз) с усиленным альтернативным промптом.
- Retry сетевых ошибок с exponential backoff (1s → 2s → 4s), таймаут запроса.
- Ожидание загрузки модели (обработка ответа `503 Loading model`).
- Прерываемая обработка: пауза/возобновление/остановка, у стриминговых серверов стоп реагирует за пару секунд.
- Встроенные пресеты промптов + сохранение/загрузка своих в `presets.json`.
- Опциональное триггер-слово стиля, которое подставляется первой строкой каждого `.txt`.
- Прогресс-бар, статус «файл X из Y», real-time лог, статистика.
- Превью + ручные действия: Принять / Перегенерировать / Редактировать / Пропустить.
- Сохранение прогресса (`progress.json`) — можно продолжить после перезапуска.

### Работа с готовым датасетом

- **Статистика тегов**: частота каждого тега по числу файлов — видно, какие теги редкие/лишние.
- **Массовые операции над тегами** с предпросмотром и бэкапом (`.bak`): удалить тег, добавить тег, заменить тег целиком, подстроковая замена (для опечаток). Проза (описания, скобочные блоки персонажей) при этом не затрагивается.
- **Ретрофит триггер-слова** по всему датасету: добавить/убрать триггер первой строкой во всех `.txt` разом (идемпотентно).
- **Откат** последней массовой операции из `.bak`.
- **Галерея**: просмотр по одному изображению, фильтр «только без капшена» и поиск по подстроке/тегу, ручное редактирование и удаление капшенов.
- **Сводка «здоровья» датасета**: сколько картинок, капшенов, картинок без капшена, с триггером/без, доступных бэкапов.

## Установка

```
pip install -r requirements.txt
```

Требуется Python 3.10+.

## Запуск

1. Поднимите локальный **OpenAI-совместимый** сервер с загруженной **multimodal**
   (vision) моделью — например, LLaVA, Qwen2-VL, Pixtral, MiniCPM-V, Gemma 3,
   Llama 3.2 Vision. Примеры:

   - **llama.cpp**: `llama-server -m model.gguf --mmproj mmproj.gguf --port 5005`
     → API на `http://127.0.0.1:5005/v1`
   - **oobabooga**: `python server.py --api --auto-launch`
     → API на `http://127.0.0.1:5000/v1`

2. Запустите приложение:
   ```
   streamlit run app.py
   ```

3. В сайдбаре укажите **API URL** и **имя модели**, нажмите «Проверить соединение»
   (имя активной модели можно подтянуть кнопкой 🔄).
4. Выберите папку, режим, пресет/промпты и нажмите **Запустить**.

## Структура

```
app.py                 UI и оркестрация
config.py              настройки и дефолтные промпты
core/
  image_scanner.py     поиск изображений и фильтрация по режиму
  caption_client.py    клиент к серверу, base64, стриминг, retry/backoff
  worker.py            фоновый воркер обработки (start/pause/resume/stop)
  quality.py           оценка "плохого" капшена
  presets.py           встроенные + пользовательские пресеты
  dataset.py           работа с датасетом и записью .txt
  state.py             прогресс, флаги, сохранение
  logger.py            лог в файл + буфер для UI
  folder_dialog.py     системный диалог выбора папки (tkinter)
  registry.py          реестр обработанных приложением файлов
  app_settings.py      «липкие» настройки UI между сессиями
```

## Пресеты

Встроенные пресеты промптов зашиты в `core/presets.py` и доступны сразу.
Свои пресеты сохраняются в `presets.json` (этот файл в `.gitignore` — он локальный).
Пример формата — см. `presets.example.json`.

## Примечания

- Модель обязательно должна быть **multimodal** (vision), иначе изображение будет проигнорировано.
- «Пауза» останавливает после текущего файла и предлагает ручной просмотр капшена.
- Пресеты, прогресс и лог хранятся в папке приложения и не коммитятся в репозиторий.
